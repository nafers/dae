# Repo Notes

- At the end of any substantial work session, explicitly tell the user whether changes are:
  - uncommitted
  - committed locally
  - pushed to remote
- If changes are not committed, remind the user and offer to group them into sensible commits.
- If changes are committed but not pushed, remind the user and offer to push them.
- Do not assume deploys replace commits or pushes. Always call out git state separately from deploy state.
